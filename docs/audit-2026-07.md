# Switchboard — deep audit (2026-07-01)

Full-codebase audit of the Electron app (main process + renderer + workers),
followed by fixes. Every finding was traced to real code; the highest-severity
ones were re-verified by hand and, where possible, empirically (SQLite CLI for
FTS behaviour, the real `claude` binary for path encoding, a live Electron boot
for renderer/main state).

**Bottom line.** The security posture is sound: `sandbox: true` +
`contextIsolation: true` + `nodeIntegration: false` + a strong CSP (no
`unsafe-inline` scripts) + DOMPurify. No exploitable XSS, no shell injection,
no exploitable critical/high in the shipped config. The real issues were
correctness/robustness bugs affecting normal use, plus defense-in-depth gaps in
the app's own guard rails. All are addressed on `chore/remove-agent-teams`.

## Calibration note

Several subagent findings were rated HIGH; after tracing exploitability they
were downgraded — most "security" gaps are gated behind an XSS that does not
exist, or behind same-user access that already grants filesystem read. The
table's severities are the calibrated (post-verification) ones.

## Deliberately skipped (documented, not silently dropped)

- **Scheduler DST / timezone.** Cron stays LOCAL time. A spring-forward slot
  can't occur; the fire-once + advance de-dups a fall-back repeat. A `tz:` field
  was skipped as over-engineering for a personal tool. (`schedule-runner.js`)
- **Scheduler cannot run while asleep / fully quit.** Catch-up is best-effort on
  next wake/relaunch within 1h. For guaranteed schedules use `launchd`/`cron`.
- **Full transcript render virtualization.** The 25 MB read cap prevents the
  OOM/freeze; windowed rendering (load-more) is a larger renderer change, not done.

## Verified sound (positive assurance, checked)

- Electron hardening: `sandbox`/`contextIsolation`/`nodeIntegration:false`/CSP/DOMPurify.
- Shell quoting: `quoteArgvForShell` (argv arrays via `spawn`, no string concat).
- MCP auth: `crypto.randomUUID()` + `timingSafeEqual` w/ length guard + loopback + 0600 lock.
- FTS5 external-content delete/insert ordering + rowid handling — reproduced at the SQLite CLI (wrong order corrupts, code's order passes integrity-check).
- `sqlite-busy-retry` wraps every runtime writer; no cross-thread DB race (scan worker is read-only).
- Migrations bump `db_version` only after PRAGMA-checked ALTERs succeed.
- Main-process scheduler timer fires at exact cadence when backgrounded (measured: 7 ticks, <3 ms drift; not Chromium-throttled).
- No exploitable XSS: transcript/tool-output/plan/memory render paths are DOMPurify-sanitized or use `textContent`/`escapeHtml`; CSP neutralizes residual vectors.
